Role What it means in plain English Typical column names
ID The column that uniquely identifies each participant. In wide data, every row has a different ID. In long data, the same ID repeats across timepoints. Used as the random-effect grouping variable in mixed-effects models. PartID, ID, record_id, subject, participant
Group The column that says which intervention each participant received — the treatment vs the control, usually 2 or 3 levels. The main effect we want to estimate is the difference between these groups. Group, Arm, Treatment, Condition
Time In long-format data only — the column that says when each row was measured (baseline, follow-up, week N). In wide-format data, Time isn't a column; it's encoded in column suffixes like _Pre / _Post, so no column gets this role. Time, Visit, Timepoint, Session, Week, Day

Step 2bDescriptives & normality — know your data first

In plain English: Before any statistical test, look at the numbers themselves. Mean, SD, median, IQR tell you the centre and spread. Skewness / kurtosis / QQ correlation describe the raw distribution shape — useful for spotting floor/ceiling effects, outliers, or data-entry errors. Important: ANCOVA / LMM assume normal residuals, not normal raw outcomes — a skewed raw outcome often has perfectly normal residuals once Group and baseline are in the model. So these numbers are not a reason to transform or switch models yet. The real check is the residual diagnostics after fitting (Step 6); that is Usman's gate. Missing-data counts tell you whether intention-to-treat will need imputation.

ℹ Why does this matter?

This picks the statistical model the app uses. The data shape determines what's appropriate.

• Two-timepoint — each participant measured before and once after the intervention. Linear regression. Most pilot trials and small RCTs.

• Multi-timepoint — each participant measured at baseline plus 2 or more follow-up visits. Linear mixed model with Group × Time. Use when you have weekly or monthly assessments.

• Cross-over — every participant received every treatment, in random order, with washout between. Common in pharmacology and short interventions where the effect washes out.

• Dense longitudinal — wearable / sensor data with many readings per participant (e.g. heart rate every 5 minutes, daily steps). Mixed model with smooth time trends.

• Reliability — same measurement repeated to check consistency (test-retest). Kendall τ or ICC.

• Non-linear — dose-response curves, growth curves, where the relationship isn't a straight line.

The app auto-detects this from your data — override only if it's wrong.

Usually the Control arm; the Treatment is the contrast.
ℹ Why does this matter?

The reference group is the baseline of comparison. Other groups are reported as "difference from this".

• Reference = Control: contrast reads "Treatment − Control" — positive estimate means treatment scored higher.

• Reverse it and every sign flips. Same statistics, opposite-direction prose.

Pick: control / placebo / sham / standard-care arm. Doesn't change p-values, only sign conventions.

Leave this blank if you don't need any covariates — the model will just adjust for baseline. Otherwise hold Ctrl/Cmd to select multiple. Usman's adult-chiro default is "nothing beyond baseline".
ℹ Why does this matter?

Covariates are variables the model adjusts for so they don't confound the Group effect (e.g. if Treatment happens to be older on average, age can mask the real effect).

The trap: they must be pre-specified before seeing the result. Picking the covariate that gives you a small p-value is "p-hacking" — Usman insists on blinded covariate selection (Kunz & Kieser 2014).

Typical sets by study type:

• Paediatric trials (children of mixed ages and both sexes): Age + Gender — children develop fast, age is a huge driver.

• Stroke rehab trials: Time-since-stroke + Sex + Age + Affected-side — these all change how much recovery is possible.

• Trials where body size matters (gait, balance, biomechanics): Height + BMI + Age + Sex.

• Adult trials with a fairly uniform sample: usually nothing beyond the baseline of the outcome itself — adding covariates burns degrees of freedom for little gain.

If unsure: pick fewer, not more. Each covariate eats one degree of freedom and shrinks your power.

ℹ Why does this matter?

Only relevant if your data has multiple scales / subscales measured on each participant (e.g. a single questionnaire with several subscales — anxiety, depression, stress; or a balance test with multiple domains).

• One model per outcome: fit a separate model for each scale. Easy to interpret. Each result stands alone. Simpler to write up. Recommended default.

• Joint model: ONE bigger model with every scale included, sharing a participant random effect. More statistically efficient, but harder to read; a noisy scale can pull the others around.

Pick "one per outcome" unless you specifically want to test for differences between scales.

ℹ Why does this matter?

When you run several tests at once (multiple outcomes × multiple pairwise group comparisons), some will look "significant" just by chance. Correction methods inflate p-values to compensate.

• None: report raw p-values. Default for exploratory / supplementary analysis. Be transparent — state in the prose that p-values aren't corrected.

• Sidak: moderate correction. Tighter than FDR, looser than Bonferroni. Good when you have a small fixed number of tests.

• FDR (Benjamini-Hochberg): less aggressive than Sidak. Controls the proportion of false positives among rejections. Good for screening many outcomes.

• Tukey: built into many stats packages by default. Easy to compare with other tools.

Exploratory / supplementary: "none" + state it. Confirmatory claim: FDR or Sidak — decide BEFORE seeing results.

ANCOVA on post is the most powerful and removes baseline imbalance (Vickers & Altman 2001).
ℹ Why does this matter?

Four ways to handle the baseline value when comparing pre/post groups:

• ANCOVA on post (Usman's default): model the follow-up value, with baseline as a covariate. Most powerful, removes baseline imbalance (Vickers & Altman 2001 BMJ; Twisk 2018). Reports adjusted post-means.

• Change with baseline: model (post − pre), with baseline still in the model. Algebraically equivalent for the Group contrast but reports the change — clearer for clinical audiences.

• Change-only: model just (post − pre). Conservative but less powerful; re-introduces baseline imbalance. Rarely Usman's choice.

• Post-only: ignore baseline entirely. Only sensible if baseline was unreliable, irrelevant, or wasn't truly pre-randomisation.

If unsure: stick with ANCOVA on post.

ℹ Why does this matter?

The analysis approach should follow from (a) the design and (b) what the descriptive stats in Step 2b show.

• ANCOVA (recommended default for two-timepoint): parametric, includes baseline as covariate. Most powerful when residuals look roughly normal.

• ANOVA: parametric, no baseline term. Less powerful than ANCOVA; only use if there's a reason not to adjust for baseline.

• Linear mixed model (LMM): parametric with random effects for repeated measures or nested designs. Default for multi-timepoint and dense longitudinal designs.

• GLMM: for outcomes that aren't normally distributed — skewed-positive (Gamma), counts (Poisson), or yes/no (Binomial).

• Non-parametric: when QQ correlation is < 0.95 or sample size is very small. Less powerful but doesn't assume normality.

• Robust LMM: same shape as LMM but downweights extreme outliers. Useful for activity-monitor data where a few participants have wildly high values.

• Non-linear mixed model: dose-response curves, growth curves — where the relationship between predictor and outcome is curved, not straight.

Sample shows — missing.
ℹ Why does this matter?

Missing data is the #1 thing that derails trial reports. The choice has to be made transparently before looking at the result.

• Complete-case: the simplest — drop any participant with a missing value. Fine when missingness is small (< 5%) and not informative. Loses power; can bias if missingness depends on the outcome.

• Per-protocol (PP): only participants who completed the intervention as designed. Answers "does this treatment work when used correctly" but breaks randomisation balance.

• ITT with LMM: keep everyone, even partial completers. The mixed model uses all available data via full-information maximum likelihood (FIML). Recommended default for multi-timepoint trials when there's any drop-out.

• ITT with MICE: multiple imputation. Create m datasets (50 is a good number) where missing values are filled in based on what's observed, run the analysis on each, then pool. Run with a fixed random seed so the result is reproducible.

• LOCF: carry forward last observation. Historical, biased in most directions — avoid unless a regulator specifically asks for it.

Best practice: pre-specify ITT as primary, PP as sensitivity.

If the Q-Q plot in Step 2b looked bad, follow the decision diagram and pick a transform here.
ℹ Why does this matter?

When the outcome's residuals aren't normal, transforming the outcome before fitting can make the model assumptions hold without changing the design.

• log(Y): when the outcome is strictly positive and has a long right tail (e.g. cortisol, hospital LOS, heart-rate variability). Interpret coefficients as multiplicative.

• log(Y + c): same idea, but adds a small offset so zeros don't crash. Typical c = 1, or the smallest non-zero / 2.

• √Y: milder than log. Good for counts (step counts, headache frequencies). Easier to back-transform than log.

• 1/Y: strong correction. Rarely needed unless distribution is extreme.

Transform = fit a different model on the transformed scale. State the transformation in the methods. Report back-transformed means if your audience expects them in original units.

Leave blank to use the count of unique participant IDs.

Model diagnostics

    

      How to read these three plots
      

        
① Fitted vs residuals. Points should scatter randomly around the zero line with no funnel shape. Funnel = bad (variance grows with prediction — try log-transform). Curved pattern = bad (you're missing a non-linear term).

        
② QQ plot. Points should sit on the dashed diagonal. Bowed-out tails mean the residuals are heavier than a normal distribution. Mild deviation is fine; severe S-curves mean the model's normality assumption is broken.

        
③ Histogram of residuals. Should look roughly bell-shaped, centred at zero. Skew, bimodality, or heavy tails are warning signs. Usman's rule: if a diagnostic looks bad, say so in the prose — don't ship a bad-fit model silently.
