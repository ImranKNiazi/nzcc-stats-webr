webR proof-of-concept

Runs real R (lme4 + emmeans) entirely in your browser — no R install, no server, and your data never leaves this machine. This proves the engine path for the NZCC Stats App's multi-timepoint designs.

How to run this file: webR needs the page to be served with cross-origin-isolation headers — a plain file:// open or python -m http.server will fail with a URL error. Use the included serve.py (same folder), which sends those headers:
  1. Open a terminal in this folder.
  2. Run python3 serve.py
  3. Open http://localhost:8000/webR-POC-multitimepoint.html
First run downloads webR + lme4 (tens of MB, a minute or two); cached afterwards. Best in Chrome or Edge.
What it will do
  1. Boot webR (R 4.6) in the browser.
  2. install.packages(c("lme4","emmeans")) from the WebAssembly repo.
  3. Build a multi-timepoint sample (20 participants × 3 visits × 2 arms — the design the current in-browser engine refuses).
  4. Fit Usman's mixed model lmer(value ~ baseline + group*time + (1|id)) and report emmeans pairwise Group contrasts at each visit.
